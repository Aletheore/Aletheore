I got tired of code-review tools that just guess. So I built one that has to show its work.

Aletheore is a deterministic repository scanner: no LLM in the scan path, fully unit-tested, and every downstream claim traces back to specific evidence in your repo.

Start Your First Audit Read the Protocol

Why deterministic-first.

No LLM ever runs in the scan path. Aletheore reads your repository with tree-sitter and git log, writes what it found to evidence.json, and every other command - query, diff, the dashboard, the MCP server, even audit's written report - reads from that same file. Nothing downstream states a claim it can't point back to a specific field in it.

How it actually works.

aletheore scan runs once and writes two files: evidence.json (the canonical record - languages, dependency graph, clusters, git activity, secrets, licenses, vulnerabilities, API endpoints) and evidence.toon, a TOON-encoded copy of the same data specifically for the MCP server's tool results and the coding-agent adapter audit uses - because that's where token cost is actually paid, not in the file Aletheore itself keeps on disk.
